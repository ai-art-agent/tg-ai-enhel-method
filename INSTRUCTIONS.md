# Запуск от А до Я (для новичков)

Этот раздел — пошаговый запуск бота **с нуля**, для тех, кто никогда не устанавливал Python и не запускал программы из командной строки. Выполняйте пункты по порядку.

---

### Шаг 0. Что должно быть под рукой

- **Компьютер** с Windows 10 или 11 и доступом в интернет.
- **Учётная запись Telegram** (приложение на телефоне или на ПК, или web.telegram.org).
- **Папка с ботом** — та, где лежат файлы `bot.py`, `requirements.txt`, `INSTRUCTIONS.md` и т.д. Запомните или запишите полный путь к ней, например: `C:\Users\ИмяПользователя\telegram-ai-psychologist`.

---

### Шаг 1. Установка Python

Бот написан на языке Python. Сначала нужно установить сам Python.

1. Откройте сайт: **https://www.python.org/downloads/**
2. Нажмите большую жёлтую кнопку **«Download Python 3.x.x»** (актуальная версия).
3. Запустите скачанный установщик (`python-3.x.x-amd64.exe`).
4. **Важно:** в самом начале установки поставьте галочку **«Add python.exe to PATH»** (внизу окна). Без этого команды `python` и `pip` не будут находиться в системе.
5. Нажмите **«Install Now»** и дождитесь окончания установки. Закройте установщик кнопкой **«Close»**.
6. **Проверка:** нажмите на клавиатуре `Win + R`, введите `cmd` и нажмите Enter. Откроется чёрное окно — это **командная строка**. Введите команду:
   ```text
   python --version
   ```
   и нажмите Enter. Должно появиться что-то вроде `Python 3.12.x`. Если вместо этого пишет «не является внутренней или внешней командой» — вернитесь к п. 4 и переустановите Python, обязательно включив «Add to PATH».

---

### Шаг 2. Открытие папки с ботом в командной строке

Нужно «перейти» в папку, где лежит бот, чтобы все следующие команды выполнялись именно там.

1. Откройте **Проводник** (Win + E) и перейдите в папку с ботом (например `telegram-ai-psychologist`).
2. В адресной строке проводника (вверху, где путь вроде `C:\Users\...`) **кликните по пути один раз** — он выделится. Нажмите Ctrl+C, чтобы скопировать путь.
3. Снова откройте командную строку: `Win + R` → введите `cmd` → Enter.
4. Введите команду **смены папки** (подставьте свой скопированный путь вместо примера):
   ```text
   cd C:\Users\AI_Art\telegram-ai-psychologist
   ```
   Если путь содержит пробелы, возьмите его в кавычки:
   ```text
   cd "C:\Users\Имя Пользователя\telegram-ai-psychologist"
   ```
5. Нажмите Enter. Строка приглашения должна измениться и показывать вашу папку. Теперь вы «находитесь» в папке проекта.

**Альтернатива:** в Проводнике откройте папку с ботом, в адресной строке введите `cmd` и нажмите Enter — командная строка откроется сразу в этой папке.

---

### Шаг 3. Установка зависимостей (библиотек)

Бот использует готовые библиотеки: для Telegram и для работы с DeepSeek. Их нужно установить один раз.

1. Убедитесь, что командная строка открыта **в папке с ботом** (см. шаг 2).
2. Введите команду:
   ```text
   pip install -r requirements.txt
   ```
3. Нажмите Enter. Начнётся загрузка и установка пакетов — подождите. В конце не должно быть сообщений об ошибках (красным). Обычно в последней строке пишут что-то вроде «Successfully installed ...».
4. Если появилась ошибка **«pip не найден»** или **«python не найден»** — значит, Python не добавлен в PATH (вернитесь к шагу 1, п. 4). Можно попробовать вместо `pip` набрать `py -m pip install -r requirements.txt`.

---

### Шаг 4. Создание файла с ключами (.env)

Бот не должен «зашивать» токены в код. Все секреты хранятся в одном файле с именем **`.env`** в папке с ботом.

**Что такое .env:** это обычный текстовый файл. Имя файла — именно `.env` (точка в начале, без имени перед ней). Внутри — строки вида `ПЕРЕМЕННАЯ=значение`.

**Как создать .env в Windows:**

**Способ А (Блокнот):**

1. Откройте **Блокнот** (Notepad).
2. Введите (подставьте свои токен и ключ):
   ```text
   TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
   DEEPSEEK_API_KEY=ваш_ключ_DeepSeek
   ```
   Без пробелов вокруг `=`, каждое значение в своей строке. Кавычки не нужны.
3. Меню **Файл → Сохранить как**.
4. В поле «Имя файла» напишите: **`.env`** (с точкой в начале).
5. В «Тип файла» выберите **«Все файлы (*.*)»**, иначе Windows добавит расширение `.txt`.
6. Убедитесь, что сохраняете **в папку с ботом** (та же папка, где `bot.py`). Нажмите «Сохранить».

**Способ Б (если уже есть .env.example):**

1. Откройте папку с ботом в Проводнике.
2. Найдите файл **`.env.example`**. Скопируйте его (Ctrl+C) и вставьте в ту же папку (Ctrl+V).
3. Переименуйте копию в **`.env`** (удалите часть `.example`).
4. Откройте `.env` в Блокноте и замените placeholder-ы на свои **TELEGRAM_BOT_TOKEN** и **DEEPSEEK_API_KEY**, сохраните файл.

**Где взять токен и ключ:**  
- Токен бота — у @BotFather в Telegram (см. «Этап 1» ниже).  
- Ключ DeepSeek — в личном кабинете на platform.deepseek.com (см. «Этап 2»).

### Шаг 5. Запуск бота

1. Снова откройте командную строку **в папке с ботом** (как в шаге 2).
2. Введите:
   ```text
   python bot.py
   ```
   и нажмите Enter.
3. Если всё настроено верно, появится строка:
   ```text
   Бот запущен. Остановка: Ctrl+C
   ```
   Ошибок быть не должно. Окно **не закрывайте** — пока оно открыто, бот работает.
4. Откройте Telegram, найдите своего бота по @username и нажмите **«Запустить»** или отправьте `/start`. Бот должен ответить.
5. Чтобы **остановить** бота, вернитесь в окно командной строки и нажмите **Ctrl+C**. После этого бот перестанет отвечать, пока вы снова не запустите `python bot.py`.

**Если при запуске появилась ошибка:**

- **«В .env не указан TELEGRAM_BOT_TOKEN»** или **«DEEPSEEK_API_KEY»** — проверьте файл `.env`: он должен лежать в той же папке, что и `bot.py`, и содержать ровно такие имена переменных и ваши значения без лишних пробелов и кавычек.
- **«python не является внутренней или внешней командой»** — Python не в PATH; переустановите Python с галочкой «Add to PATH» (шаг 1).
- **«No module named 'telegram'»** (или другое «No module named ...»)** — зависимости не установлены; повторите шаг 3 из папки с ботом.

---

### Шаг 6. Запуск двойным кликом (Windows, по желанию)

Чтобы не открывать командную строку каждый раз, можно запускать бота двойным кликом по файлу **`start_bot.bat`**.

1. Убедитесь, что бот хотя бы раз успешно запускался через `python bot.py` (шаг 5).
2. В папке с ботом найдите файл **`start_bot.bat`**.
3. Дважды кликните по нему. Откроется окно с текстом «Бот запущен...» — бот работает. Закрытие окна останавливает бота.

Если при двойном клике окно сразу закрывается с ошибкой — откройте `start_bot.bat` через «Изменить» (правый клик) и проверьте, что внутри есть строка `python bot.py`, а путь к папке правильный. Либо запускайте бота вручную через командную строку (шаг 5).

**Частые вопросы:**

- **Бот не отвечает в Telegram.** Убедитесь, что окно с запущенным ботом не закрыто и в нём написано «Бот запущен. Остановка: Ctrl+C». Если окно закрыли — запустите снова `python bot.py` или `start_bot.bat`.
- **Где должна лежать папка с ботом?** Где угодно: Рабочий стол, Документы, отдельная папка на диске. Главное — запомнить путь и открывать командную строку именно в этой папке (шаг 2).

## Этап 1: Регистрация бота в Telegram

### Что сделать

1. Откройте Telegram (приложение или web.telegram.org).
2. В поиске введите **@BotFather** и откройте чат.
3. Отправьте команду: `/newbot`.
4. BotFather попросит ввести:
   - **Имя бота** — как его будут видеть пользователи (например: «ИИ-психолог»).

5. После создания BotFather пришлёт **токен** — длинную строку вида `7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`.  
   **Сохраните токен:** он понадобится для файла `.env`.

### Что сделать

   Обычно ключ выглядит как длинная строка символов.

**Важно:** DeepSeek API совместим с форматом OpenAI — в коде используется библиотека `openai` с указанием `base_url` и вашего ключа DeepSeek.

Подробное создание файла с нуля описано в разделе **«Запуск от А до Я» → Шаг 4**.

В корне проекта создайте файл **`.env`** (имя файла — именно `.env`, без имени перед точкой). Пример содержимого:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
DEEPSEEK_API_KEY=ваш_ключ_DeepSeek
```

Опционально: `DEEPSEEK_API_KEY_VALIDATOR=второй_ключ` — отдельный ключ для валидатора (подробно в разделе «Два ключа DeepSeek» ниже).

- Не публикуйте `.env` в интернете и не добавляйте в репозиторий (должен быть в `.gitignore`).
- Файл `.env.example` можно хранить в репозитории как шаблон без секретов.

### Два ключа DeepSeek (опционально, рекомендуется при нагрузке)

Ответы психолога и проверка валидатором идут в DeepSeek API. По умолчанию оба запроса используют один ключ `DEEPSEEK_API_KEY`. Лимиты (запросов в минуту, токенов в минуту) при этом общие: при наплыве пользователей можно упереться в лимит.

**Зачем отдельный ключ для валидатора:** если задать второй ключ `DEEPSEEK_API_KEY_VALIDATOR`, валидатор будет ходить в API уже по нему. В итоге:
- лимиты разделены — валидатор не «съедает» квоту ответов психолога;
- при проблемах по одному ключу (исчерпан баланс, rate limit) второй может продолжать работать;
- удобнее учитывать расход отдельно (если ключи от разных аккаунтов или с разными лимитами).

**Что сделать по шагам (на своём компьютере или на ВМ — одинаково):**

1. **Получить второй API-ключ DeepSeek**
   - Откройте в браузере: **https://platform.deepseek.com** (или актуальный адрес личного кабинета DeepSeek).
   - Войдите в тот же аккаунт, с которого берёте основной ключ (или создайте второй аккаунт, если нужна отдельная квота).
   - Перейдите в раздел выдачи API-ключей (обычно «API Keys», «Ключи» или «Settings»).
   - Создайте **новый ключ** (Create new key / Создать ключ). Скопируйте его и сохраните в надёжное место — второй раз он может не показываться полностью.

2. **Добавить ключ в `.env`**
   - Откройте файл **`.env`** в папке с ботом (Блокнот, Notepad++, VS Code и т.п.).
   - Добавьте строку (подставьте свой второй ключ вместо `второй_ключ_DeepSeek`):
     ```env
     DEEPSEEK_API_KEY_VALIDATOR=второй_ключ_DeepSeek
     ```
   - Сохраните файл. Не ставьте пробелы вокруг `=`, не заключайте ключ в кавычки. Остальные переменные (`TELEGRAM_BOT_TOKEN`, `DEEPSEEK_API_KEY` и т.д.) не трогайте.

3. **Перезапустить бота**
   - Если бот запущен локально: остановите его (`Ctrl+C` в окне, где он работает) и снова запустите (`python bot.py` или двойной клик по `start_bot.bat`).
   - Если бот запущен как служба на ВМ: выполните перезапуск службы (см. ниже раздел «Что сделать на ВМ»).

4. **Проверить, что второй ключ используется**
   - В логах при старте бота должно появиться сообщение:  
     `Валидатор использует отдельный API-ключ (DEEPSEEK_API_KEY_VALIDATOR).`  
   - Если этой строки нет — проверьте, что в `.env` точно указано `DEEPSEEK_API_KEY_VALIDATOR=...` без опечаток и что бот перезапущен после изменения `.env`.

**Если второй ключ не задавать:** ничего дополнительно делать не нужно. Бот работает как раньше: валидатор использует `DEEPSEEK_API_KEY`. Добавлять `DEEPSEEK_API_KEY_VALIDATOR` не обязательно.

### Что сделать на ВМ (если бот развёрнут на сервере / виртуальной машине)

Если вы уже создали `.env` на ВМ и хотите подключить второй ключ DeepSeek:

1. **Подключитесь к ВМ по SSH** (как обычно, например: `ssh пользователь@IP_адрес`).

2. **Откройте файл `.env` на ВМ:**
   ```bash
   cd ~/tg-ai-enhel-method
   nano .env
   ```
   (Вместо `~/tg-ai-enhel-method` подставьте путь к папке с ботом на ВМ, если он другой.)

3. **Добавьте строку с вторым ключом** (в любом месте файла, например после `DEEPSEEK_API_KEY=...`):
   ```env
   DEEPSEEK_API_KEY_VALIDATOR=ваш_второй_ключ_DeepSeek
   ```
   Сохраните в nano: `Ctrl+O`, Enter, выход: `Ctrl+X`.

4. **Перезапустите службу бота**, чтобы подхватить новый ключ:
   ```bash
   sudo systemctl restart tg-ai-enhel-method
   ```
   (Имя службы может отличаться — смотрите, как она названа у вас в DEPLOY_YANDEX_CLOUD.md или в `systemctl list-units`.)

5. **Проверьте логи** — при старте должна появиться строка про отдельный ключ валидатора:
   ```bash
   journalctl -u tg-ai-enhel-method -n 30 --no-pager
   ```
   Ищите: `Валидатор использует отдельный API-ключ (DEEPSEEK_API_KEY_VALIDATOR).`

6. **Права на `.env`** (рекомендуется): чтобы только владелец мог читать файл с ключами:
   ```bash
   chmod 600 .env
   ```

Если после добавления ключа бот не стартует — проверьте, что в `.env` нет лишних пробелов, переносов в середине значения и что ключ скопирован полностью.

## Запуск бота

Пошаговый запуск с нуля (установка Python, командная строка, создание .env) — в разделе **«Запуск от А до Я»** в начале инструкции.

Кратко:
1. Установка зависимостей (один раз): `pip install -r requirements.txt`
2. Заполните `.env` (токен и ключ DeepSeek).
3. Запуск:
   - Из терминала в папке с ботом: `python bot.py`
   - В Windows: двойной клик по `start_bot.bat`

Остановка: в окне с ботом нажмите `Ctrl+C`.

После заполнения ответов на этапах 1–6 вы можете передать их для финальной подстройки скрипта и текстов в `bot.py`.
